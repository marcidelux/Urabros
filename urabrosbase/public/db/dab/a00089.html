Detailed Description

OutgoingMessageHandler is responsible for handling the outgoing message queue.

Author
Marton.Lorinczi
Date
Sep 14, 2020
    Only the UrabrosMaster.c is allowed to use the @see uMsgOutPop() and @see uMsgSend() functions.\n
    The @see uMsgOutPut() can be used from uTasks. But for easier use there are some gigher level functions for\n
    putting a message to the outgoing buffer.
    The outgoing buffer is a static variable of this source file, so it's a private variable.
    UrabrosMaster.c one of it's field so an extern variable #uMsgOutWaitingNumPtr was created for that purpose.

◆ uMsgOutPop()

Urabros_MsgStatus uMsgOutPop ( Urabros_MsgPtr  uMsgPtr)

Pops the last message from the uOutgoinggBuffer.

Parameters
uMsgPtrThis is a pointer to an Urabros_Msg variable.
If the pop sucseeded than it will load the correct values to the pointer variable.
If the queue was empty it will not make any modifications on the pointed variable.
Returns
Returns uMsg_Ok if it went well, returns uMsg_BufferIsEmpty if the queue was empty

◆ uMsgOutPut()

Urabros_MsgStatus uMsgOutPut ( Urabros_MsgPtr  uMsgPtr)

Puts an Urabros_Msg tot the outgoing queue.

Parameters
uMsgPtrThe pointer to the message we want to put into the queue. @returnReturns uMsg_Ok if it went well, returns uMsg_BufferIsFull if the queue was full. If this happens oftenly than the processing time is not fast enough.

Puts an Urabros_Msg tot the outgoing queue.

There was a request from a user to make a more friendly function what can place a buffer to the outgoing message, without using crc calculation and other stuffs. This method needed multiple functions from multiple files, and avoiding too much header includes, I just used the extern keyword, if you come up with a better solution dont hesitate to tell it to the owner of the framework :)
